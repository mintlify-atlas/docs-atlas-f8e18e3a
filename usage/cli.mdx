## Installation

Install `js-beautify` globally to make the CLI tools available on your `PATH`:

```bash
npm -g install js-beautify
```

This provides three executables: `js-beautify`, `css-beautify`, and `html-beautify`.

## Common use cases

**Beautify a file to stdout**

```bash
js-beautify foo.js
```

**Write in-place, replacing the original file**

```bash
js-beautify --replace foo.js
```

**Read from stdin**

```bash
echo 'var x={a:1}' | js-beautify -f -
```

**Write output to a different file**

```bash
js-beautify -o foo.pretty.js foo.js
```

**Use a config file**

```bash
js-beautify --config .jsbeautifyrc foo.js
```


## CSS and HTML as an alternative invocation

You can also invoke the CSS and HTML beautifiers through `js-beautify` by passing a `--type` flag:

```bash
js-beautify --css styles.css
js-beautify --html index.html
```

<Tip>
The `--type` flag selects which beautifier runs — it does **not** filter which files are processed. Use `css-beautify` or `html-beautify` directly for clarity.
</Tip>

## Loading settings from environment or config file

In addition to CLI arguments, you can provide configuration through:

- Any `jsbeautify_`-prefixed environment variable
- A JSON-formatted file passed via `--config`
- A `.jsbeautifyrc` file containing JSON at any level of the filesystem above `$PWD`

Configuration sources earlier in this list take precedence over later ones.
